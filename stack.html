Тут перечислены непосредственно активно использующиеся технологии и решения в данный момент, в остальном их спектр не ограничен ничем кроме необходимости внедрения и времени на это.
Инфраструктура:
  • Linux(deblike/centos)
  • Python, C++, bash
  • Nginx, Named
  • Стараюсь избегать без острой необходимости использования баз данных и другого дополнительного ПО, но если нужда возникает то предпочитаю MongoDB, MySQL, PostgreSQL
  • Менеджеры очередей и подобное тоже использую самописные, если требуется готовое решение - RabbitMQ
  • По необходимости хранилища ключ-значение - Redis, Memcached
  • QEMU, KVM, libvirt
  • sshfs, git
  • Если требуется ПО для командной разработки - GitLab

    • Часто используемые библиотеки Python:
  • plotly, bokeh, streamlit, matplotlib, PIL
  • numpy, pandas, json
  • multiprocessing, subprocess, os, shutils, glob
  • Библиотеки для API мессенджеров(discord, telegram)
  • CCXT, unicorn-binance-websocket-api, requests, другие готовые библиотеки для взаимодействия с биржами если имеются.
  • Большое количество других библиотек, нужных по ситуации.

    • В проектировании систем предпочитаю:
  • Отсутствие лишнего стороннего ПО, если это возможно, простоту инфраструктуры для уменьшения потенциальных точек отказа
  • Модульные решения с гибкостью замены составных частей
  • Если есть возможность создать систему легко переносимым монолитом без ущерба работе и поддержке, с минимумом дополнительных сущностей - предпочту этот вариант
  • Предпочитаю скорость и работу на низком уровне пользовательским интерфейсам и юзабилити, хотя второе не отрицаю и реализую где это не вредит производительности и занимает разумное время
  • Информирование в discord/telegram с кастомными метриками и алертами, в том числе некоторые логи, чтобы быть максимально в курсе без необходимости заходить по ssh/на веб страницы
